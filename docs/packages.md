# Package Manager

Jerboa includes a semver-based package manager for declaring, resolving,
and locking dependencies between Jerboa/Chez Scheme libraries.

**Import:** `(jerboa pkg)`

---

## Package Manifest

A package is described by its name, version, and dependencies.

```scheme
(make-package name version deps description)
;; name:        symbol
;; version:     version record (see below)
;; deps:        list of dep-spec records
;; description: string

(package? x)
(package-name pkg)
(package-version pkg)
(package-dependencies pkg)
(package-description pkg)
```

---

## Versions

Versions follow semantic versioning: `major.minor.patch`.

```scheme
(make-version major minor patch)
(version? x)
(version-major v)
(version-minor v)
(version-patch v)
(version->string v)       ; => "1.2.3"
(string->version "1.2.3") ; => #<version 1.2.3>

(version<? v1 v2)         ; #t if v1 < v2
(version=? v1 v2)         ; #t if v1 = v2
(version-satisfies? v constraint)
;; constraint examples:
;;   "^1.2.0"   — compatible (same major, >= minor.patch)
;;   "~1.2.0"   — patch compatible (same major.minor, >= patch)
;;   ">=1.0.0"  — minimum version
;;   "1.2.3"    — exact version
```

### Constraint semantics

| Constraint | Meaning |
|------------|---------|
| `^1.2.3` | `>=1.2.3 <2.0.0` — compatible with 1.x |
| `~1.2.3` | `>=1.2.3 <1.3.0` — patch updates only |
| `>=1.0.0` | at least 1.0.0 |
| `1.2.3` | exactly 1.2.3 |

```scheme
(version-satisfies? (string->version "1.3.0") "^1.2.0")   ; => #t
(version-satisfies? (string->version "2.0.0") "^1.2.0")   ; => #f
(version-satisfies? (string->version "1.2.5") "~1.2.0")   ; => #t
(version-satisfies? (string->version "1.3.0") "~1.2.0")   ; => #f
```

## Dependency Specs

```scheme
(make-dep-spec name constraint source)
;; name:       package name (symbol)
;; constraint: version constraint string
;; source:     'registry | "https://..." | "file:../path"

(dep-spec? x)
(dep-spec-name ds)
(dep-spec-constraint ds)
(dep-spec-source ds)
```

## Package Registry

The registry maps package names to available versions.

```scheme
(make-registry)
(registry? x)
(registry-add! reg pkg)          ; add a package
(registry-find reg name version) ; find exact version; returns pkg or #f
(registry-list reg name)         ; list all versions of a package
```

```scheme
(define reg (make-registry))

(registry-add! reg
  (make-package 'web-server
    (string->version "1.0.0")
    (list (make-dep-spec 'http-parser "^2.0.0" 'registry))
    "A simple web server"))

(registry-add! reg
  (make-package 'web-server
    (string->version "1.1.0")
    (list (make-dep-spec 'http-parser "^2.1.0" 'registry))
    "A simple web server"))

(registry-list reg 'web-server)
; => (#<package web-server 1.0.0> #<package web-server 1.1.0>)
```

## Dependency Resolution

```scheme
(resolve-dependencies registry root-deps)
;; root-deps: list of dep-specs
;; Returns: list of package records in topological order (deps before dependents)

(dependency-graph registry deps)
;; Returns: alist of (name . list-of-dep-names) for the full transitive closure

(topological-sort graph)
;; graph: alist of (node . (dep-node ...))
;; Returns: list of nodes in dependency order (leaves first)
```

```scheme
(define deps
  (list (make-dep-spec 'web-server "^1.0.0" 'registry)
        (make-dep-spec 'database   "~2.3.0" 'registry)))

(define resolved (resolve-dependencies reg deps))
;; resolved is a list of packages in build order
(for-each (lambda (pkg)
            (printf "~a ~a~%"
              (package-name pkg)
              (version->string (package-version pkg))))
          resolved)
```

## Lock File

Lock files record the exact resolved versions to ensure reproducible builds.

```scheme
(make-lock-file packages)   ; packages: list of (name . version-string) pairs
(lock-file? x)

(lock-file-write lf path)   ; write lock file to path
(lock-file-read path)       ; read lock file from path; returns lock-file record
```

### Lock file format (`jerboa.lock`)

```scheme
;; Generated by jerboa pkg — do not edit
((web-server . "1.0.3")
 (http-parser . "2.1.4")
 (database . "2.3.1"))
```

## Package File

The `jerboa.pkg` file declares a package's metadata.

```scheme
(read-package-file path)    ; read jerboa.pkg → package record
(write-package-file pkg path) ; write package record → jerboa.pkg
```

### `jerboa.pkg` format

```scheme
(package
  (name my-app)
  (version "1.0.0")
  (description "My application")
  (dependencies
    (web-server "^1.0.0" registry)
    (database   "~2.3.0" registry)
    (utils      ">=0.5.0" "https://github.com/user/utils")))
```

## Complete Workflow

```scheme
(import (chezscheme) (jerboa pkg))

;; 1. Create a registry with available packages
(define reg (make-registry))
(for-each (lambda (pkg) (registry-add! reg pkg))
          (load-packages-from-server "https://packages.jerboa-lang.org"))

;; 2. Read this project's package file
(define this-pkg (read-package-file "jerboa.pkg"))

;; 3. Resolve dependencies
(define resolved
  (resolve-dependencies reg (package-dependencies this-pkg)))

(printf "Resolved ~a dependencies:~%" (length resolved))
(for-each (lambda (p)
            (printf "  ~a ~a~%"
              (package-name p)
              (version->string (package-version p))))
          resolved)

;; 4. Write lock file
(lock-file-write
  (make-lock-file
    (map (lambda (p)
           (cons (package-name p)
                 (version->string (package-version p))))
         resolved))
  "jerboa.lock")

(printf "Lock file written.~%")
```
